## 摘要
自主编码智能体的日益普及加速了软件开发，但也带来了在高影响文件路径范围内的安全风险，这些风险可能超出传统人工审查的能力。尽管先前研究主要从功能正确性和生产力方面评估这些系统，本文利用 AIDev 数据集进行大规模实证研究，系统性地表征智能体生成的拉取请求（PRs）中的安全代码异味。通过结合经验证的 LLM 评判框架和人工定性分析，我们识别并分类了跨越 4,022 个拉取请求的 16,112 次文件修改中的安全配置错误。结果表明，38.9% 的智能体生成的拉取请求至少包含一种安全异味，其中供应链完整性问题占所有检测到的安全异味的 82.3%。此外，硬编码凭证占所有严重级别安全异味的 99.6%。关键的是，我们发现，在这些智能体辅助的工作流中，67.6% 的真实泄露秘密是由人类协作者引入的，而现有的自动化和人工审查流程在集成前未能检测出 81.1% 的这些凭证。这些发现突显了智能体辅助软件开发工作流中的重大安全风险，并暗示开发者警惕性可能下降。它们还强调了在人机协作环节直接实施上下文感知安全护栏的迫切需求。

## Abstract
The increasing adoption of autonomous coding agents accelerates software development but also introduces scoped security risks within high-impact file paths that can outpace traditional human review capacity. While prior research has primarily evaluated these systems in terms of functional correctness and productivity, this paper presents a large-scale empirical study using the AIDev dataset to systematically characterize security code smells in agent-generated pull requests (PRs). Through a combination of a validated LLM-as-a-judge framework and manual qualitative analysis, we identify and classify security misconfigurations across 16,112 file changes spanning 4,022 pull requests. Our results reveal that 38.9% of agent-generated PRs contain at least one security smell, with supply chain integrity issues accounting for 82.3% of all detected security smells. Furthermore, hard-coded credentials constitute 99.6% of all critical-severity security smells. Crucially, we find that human collaborators are responsible for introducing 67.6% of genuine leaked secrets within these agent-assisted workflows, while existing automated and human review processes fail to detect 81.1% of these credentials prior to integration. These findings highlight substantial security risks in agent-assisted software development workflows and suggest a potential reduction in developer vigilance. They also underscore the urgent need for context-aware security guardrails implemented directly at the point of human-AI collaboration.

- **研究动机**：自主编码智能体（如 GitHub Copilot、Devin、Cursor 等）正在大幅提升软件开发速度，但生成的大量 Pull Request（PR）可能超出人工审查能力，导致安全风险被忽略。
- **核心问题**：现有研究多关注智能体生成代码的功能正确性，缺乏对其引入的 **结构性安全异味**（security smells，即潜在安全问题模式）的大规模实证分析。
- **整体含义**：论文旨在揭示人机协作流程中 **“安全债务”** 的分布、来源与审查失效情况，为构建上下文感知的安全护栏提供依据。

## 方法论
- **核心思想**：将安全异味检测转化为一个基于 LLM 的分类任务，结合人工验证，对大规模真实 PR 中的代码改动进行安全风险表征。
- **技术流程**：
  1. **语料构建**：从 AIDev 数据集中筛选高风险文件路径（如 Workflow、Dockerfile、配置文件、Shell 脚本等），仅保留新增行（+行），最终获得 16,370 处文件改动，覆盖 4,064 个 PR。
  2. **安全异味分类体系**：基于 OWASP、CIS 基准和 GitHub 安全实践，定义六类异味：
     - `secrets_identity`：硬编码密钥、令牌等
     - `cleartext_transport`：明文传输、弱加密
     - `over_privilege_execution`：过高权限、root 容器等
     - `permissive_network`：全接口绑定、公开访问
     - `misconfig_hardening`：调试模式、日志记录密钥等
     - `supply_chain_integrity`：可变标签、未固定版本安装
  3. **LLM-as-a-judge 检测**：使用两个开源模型（Qwen3.6-35B-A3B-FP8 和 Gemma-4-26B-A4B-IT-FP8）以零样本方式对每个文件改动独立判断，输出是否存在异味、类别、行号、严重级别（Critical/Major/Minor）和理由。以联合方式合并两模型结果（至少一个标记则为脏文件）。
  4. **人工验证与一致性评估**：从 376 个文件改动（置信度 95%、误差 5%）中，由两位作者独立标注，第三作者仲裁，计算精确率、召回率、F1 值和 Cohen’s kappa。
  5. **分层分析**：按智能体类型、仓库语言、文件类别、PR 大小对异味检出率进行拆解。
  6. **硬编码秘密深入验证（RQ2）**：对 `secrets_identity` 类别的所有标记项进行人工审查，判断是否为真实凭证、提交者是人类还是智能体、审查是否在合并前检测到。

- **分析维度**：

## 主要结论与发现
- **安全异味普遍存在**：38.9% 的智能体生成 PR 至少包含一种安全异味；文件级异味率为 23.6%。
- **供应链完整性问题占主导**：82.3% 的异味属于 `supply_chain_integrity`（如使用可变标签、未固定版本），主要体现在 GitHub Actions 工作流和 Dockerfile 中。
- **硬编码凭证是严重异味的核心**：99.6% 的 Critical 级异味来自 `secrets_identity`。
- **人类协作者是泄露秘密的主要来源**：67.6% 的真实凭证由人类提交，而非智能体。
- **审查机制严重失效**：81.1% 的真实凭证在合并前未被任何自动化工具或人工评论捕获；仅有极少数经 GitHub Advanced Security、GitGuardian 等工具报告。
- **异味率随 PR 规模上升**：从最小 PR 的 16.2% 到最大 PR 的 53.6%，增长 37.4 个百分点。
- **不同智能体与语言差异明显**：Copilot 生成的 PR 异味率最高（45.5%），JavaScript 仓库异味率最高（55.3%），Python 最低（31.8%）。

## 优点
- **首创性实证**：首次对智能体生成 PR 进行大规模安全异味量化分析，填补了研究空白。
- **方法严谨**：LLM-as-a-judge 与人工双重验证，可靠性指标齐全；六类异味分类有据可依，可操作性强。
- **深度追踪**：对硬编码秘密深入追溯至提交者和审查行为，揭示了“人”而非“智能体”是主要风险源，为干预设计提供了关键洞见。
- **可复现性**：提供完整复制包，包含数据、脚本与文档。

## 不足与局限
- **LLM 检测召回不足**：召回率仅 0.775，约 22.5% 的异味被遗漏，导致报告的异味流行率可能被低估。
- **分析范围局限**：仅分析高风险文件路径和新增行，未涵盖删除行或低风险文件，不能全面反映仓库整体安全状况；结果不能直接推断为可被利用的漏洞。
- **外部效度受限**：基于 AIDev 公开数据，集中于特定编码智能体与开源仓库，企业环境下或有不同安全护栏，结论普适性待验证。
- **秘密验证依赖人工**：手工确认尽管细致，但存在主观偏差，且受限于 PR 可访问性（少量 PR 消失）。
- **未探究因果机制**：虽指出人类引入秘密多，但未进行行为实验解释“开发者警惕性下降”的具体成因。
