## 摘要
大型推理模型（LRMs）提升了复杂任务的性能，但也使部署时的安全控制更加困难。在黑盒环境下，防御者无法修改模型权重，只能在推理时进行干预。这种环境带来了三个实际挑战：有害意图可能被教育或角色扮演的伪装所掩盖，深度的安全分析可能引入不可忽视的延迟，以及长对抗性上下文可能稀释简单过滤器所依赖的局部线索。这些挑战可能暴露出明显的思维—输出差距，即模型在推理过程中看似谨慎，但最终仍产生不安全的回答。为了解决这一问题，我们提出了安全上下文注入（SCI），这是一种推理时框架，它将安全评估与任务生成分离，并将结构化的外部风险报告作为注入的安全上下文前置给受保护模型。该框架以两种互补的变体实现：静态模型过滤（SMF），一种轻量级的单次检测保护器，用于快速部署；以及动态智能体过滤（DAF），一种基于智能体循环的分析器，针对模糊或长上下文攻击，迭代收集和综合证据。在AdvBench和GPTFuzz上，涵盖五种越狱攻击族的基础模型和推理模型，两种变体在评估环境下均降低了攻击成功率和毒性。SMF提供了一种高效低延迟的选择，而当有害意图在语义上被伪装或分散在长上下文中时，DAF更为有效。

## Abstract
Large Reasoning Models (LRMs) improve performance on complex tasks, but they also make safety control harder at deployment time. In black-box settings, defenders cannot modify model weights and must instead intervene at inference time. This setting creates three practical challenges: harmful intent may be hidden by educational or role-play framing, deep safety analysis can introduce non-trivial latency, and long adversarial contexts can dilute the local cues that simpler filters rely on. These challenges can expose an apparent thinking--output gap, where the model appears cautious during reasoning but still produces an unsafe final answer. To address this problem, we propose Safety Context Injection (SCI), an inference-time framework that separates safety assessment from task generation and prepends a structured external risk report as injected safety context for the protected model. The framework is instantiated in two complementary variants: Static Model Filtering (SMF), a lightweight one-pass guard for fast deployment, and Dynamic Agents Filtering (DAF), an agentic-loop-based analyzer that iteratively gathers and synthesizes evidence for ambiguous or long-context attacks. Across AdvBench and GPTFuzz, spanning base and reasoning models under five jailbreak families, both variants reduce attack success rate and toxicity in the evaluated settings. SMF offers an efficient low-latency option, while DAF is more effective when harmful intent is semantically disguised or dispersed across long contexts.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

论文的核心贡献是提出了**安全上下文注入 (Safety Context Injection, SCI)** 框架，其核心思想是将安全评估与任务生成解耦，在受保护模型生成回答之前，为其注入一个结构化的外部风险报告作为额外的上下文。框架包含两个互补的防御者变体，而非级联管线。

**整体框架（SCI）**
一个外部防御者 `D` 拦截用户查询 `x`，通过安全决策函数 `Φ(x)` 进行判断：
*   若 `Φ(x) = 1`（高风险），直接返回拒绝回复 `y_refusal`。
*   若 `Φ(x) = 0`（低风险或可疑），则生成一个结构化的安全报告 `R`，并将 `R` 前置到原始查询 `x` 之前，形成增强的上下文 `R ⊕ x`，再交由受保护模型 `M_p` 生成最终回复。
*这种方法的本质是将防御方对风险的显式解释转化为受保护模型可以理解的上下文，引导其识别潜在威胁。*

**静态模型过滤 (Static Model Filtering, SMF)**
*   **核心思想**：一种轻量级、“单次通过”的防御。使用一个专用的轻量守卫模型 `M_G` 对输入进行一次推理，生成安全评估，通过确定性解析器提取标准化的安全标签和风险类别，然后根据策略决定是否阻断或注入一个简洁的安全报告。
*   **算法流程**：
    1.  守卫模型 `M_G` 对查询 `x` 评分，生成原始评估文本 `s`。
    2.  解析器 `ψ` 从 `s` 中提取安全标签 `l` (Safe/Unsafe/Controversial) 和风险类别 `C`。
    3.  若 `l = Unsafe`，则阻断并返回拒绝回复。
    4.  否则，将标准格式的报告 `R` 与原始查询 `x` 拼接 `(R ⊕ x)`，再送入受保护模型 `M_p` 生成最终回复。

**动态智能体过滤 (Dynamic Agents Filtering, DAF)**
*   **核心思想**：一种更强的防御，不使用一次性分类器，而是实例化一个由推理能力驱动的“安全分析师”智能体 `A`。该智能体在一个明确的智能体循环中运行，可自主决定调用一系列工具来收集交叉验证证据，最终合成一个内容更丰富的详细安全报告。
*   **算法流程**：
    1.  初始化证据集 `E` 为空。
    2.  进入循环，最多执行 `T_max` 步：
        *   智能体基于当前查询 `x` 和已有证据 `E` 决定下一步动作 `a_t`。
        *   若动作为“生成报告”，则跳出循环。
        *   否则，从工具集 `T`（如实体分析、模式检测、分类器等）中选择并执行工具 `τ`，获得观察结果 `o_t`。
        *   将 `o_t` 聚合到证据集 `E` 中。
    3.  循环结束后，智能体综合所有证据 `E`，生成最终的安全报告 `R` 和建议标签 `δ` (Block/Allow/Flag)。
    4.  根据 `δ` 决定是直接返回拒绝回复，还是将 `R` 注入上下文后再让受保护模型 `M_p` 生成回复。
