## 摘要
现有的自动红队测试流程常常遗漏依赖于攻击者身份、框架或多轮策略的攻击。这种覆盖不足低估了现实世界风险。我们提出角色条件对抗提示（PCAP），它根据攻击者角色和策略卡来约束对抗搜索，并运行并行的角色条件波束搜索，以发现多样化、可迁移的越狱攻击。PCAP 与底层搜索算法正交，大幅提高了攻击成功率（ASR）和提示多样性（例如，在 GPT-OSS~120B 上 ASR 从约58%提升至约97%），改善了攻击策略覆盖和多样性。

## Abstract
Existing automated red-teaming pipelines often miss attacks that depend on attacker identity, framing, or multi-turn tactics. This under-coverage underestimates real-world risk. We introduce Persona-Conditioned Adversarial Prompting (PCAP), which conditions adversarial search on attacker personas and strategy cards and runs parallel persona-conditioned beam searches to discover diverse, transferable jailbreaks. PCAP is orthogonal to the underlying search algorithm and substantially increases attack success rate (ASR) and prompt diversity (e.g., ASR on GPT-OSS~120B from $\approx58\% \rightarrow \approx97\%$), improving attack strategy coverage and diversity.

### 6. 主要结论与发现
- PCAP 在所有攻击者模型上均显著提升 ASR：例如 Llama 3.3 70B 从 57.7% → 97.3%，Granite 4.0 H Tiny 从 41.0% → 98.0%。
- 生成的可用越狱提示数量（提示产出）大幅增加（例如 Llama 3.3 70B 每目标由 0.72 增至 2.98 条）。

- PCAP 发现越狱提示所需的迭代次数更少，失败率更低。
- 提示具有高迁移性：许多对 GPT-OSS 120B 无效的提示仍能绕过专用防护栏（约 29%–53% 绕过 Llama Guard 3）。
- 相似度剪枝可在牺牲少量 ASR 的情况下削减约 40–50% 查询开销。
- 去除攻击者提示中的“目标字符串”约束可进一步提升 ASR 和效率。

### 7. 优点
- **正交且模块化**：PCAP 作为对 TAP 的角色/策略扩展，不绑定特定搜索算法，易于移植。
- **提升覆盖与多样性**：同时注入身份和策略卡，有效增加了攻击战术和语言风格的广度，更贴近真实攻击者行为。
- **实证扎实**：在多种模型规模和架构（密集、MoE）上均表现出显著、一致的提升，且通过人工评估验证了自动化评分的可靠性。
- **可解释性强**：通过策略分布分析和示例，清晰展示了 PCAP 如何引导攻击走向更多样化的战术组合。
- **开放性与可复现性**：承诺开源代码与配置，为防御性红队和对抗数据集生成提供实用工具。

### 8. 不足与局限
- **查询与计算成本较高**：并行多角色波束导致查询量显著增加，尤其对低能力攻击者模型开销陡增，虽可通过剪枝部分缓解，但未根本解决。
- **继承的 LLM 偏差**：角色生成、目标重述、评估均依赖 LLM，可能因对齐训练而系统性避开某些潜在漏洞。
- **仍需要人工审核**：自动化评分无法完全取代人工裁决，生成的提示用于训练防御时仍需人工过滤，避免模型过拟合合成伪影。
- **角色与策略的选择依赖启发式**：角色数量和策略子集的超参数选择高度依赖攻击者模型能力，缺乏自适应调优机制。
- **未涉及多轮交互**：PCAP 目前仅扩展了单轮搜索，尚未覆盖多轮、连续对话或更具代理性的攻击场景。
- **安全护栏的对抗性评估有限**：迁移实验仅考虑了两种护栏，未广泛测试当前业界主流防护措施的组合。
