## 摘要
我们提出swarm-attack，一个开源对抗测试框架，其中多个轻量级LLM代理通过共享内存、并行探索和进化优化进行协调。我们的结果共同表明，前沿模型的安全绕过和软件漏洞发现，即导致Anthropic的Mythos预览版限制发布的能力类别，可以使用消费级硬件和公开可用的模型以几乎零成本实现。我们报告两个实验。在第一个实验中，一个12亿参数模型的五个实例分别对GPT-4o和Claude Sonnet 4进行了225次越狱攻击。针对GPT-4o，群体攻击的有效危害率达到45.8%，产生了49个严重级别的违规；针对Claude Sonnet-4，尽管技术成功率为40%，但有效危害率为0%。在第二个实验中，相同的模型对包含9个植入CWE的易受攻击的C应用程序进行了组合的源代码分析和二进制模糊测试。借助手工编制的exploit种子语料库、正则表达式模式检测和基于AddressSanitizer的崩溃分类，该流程在消费级MacBook上大约四分钟内恢复了9个漏洞中的9个（100%召回率）。在禁用这些支架组件的情况下，同一模型通过崩溃验证恢复了0个漏洞，通过引用恢复了2个漏洞。因此，导致Anthropic的Mythos预览版限制发布的能力类别可以以几乎零成本复现；重要的推动因素是系统支架本身，它弥补了单个小型模型有限的推理能力。

## Abstract
We present swarm-attack, an open-source adversarial testing framework in which multiple lightweight LLM agents coordinate through shared memory, parallel exploration, and evolutionary optimization. Together, our results demonstrate that both safety bypass of frontier models and software vulnerability discovery, i.e., the capability class that motivated restricted release of Anthropic's Mythos Preview, are achievable at effectively zero cost using commodity hardware and openly available models. We report two experiments. In the first, five instances of a 1.2 billion parameter model conducted 225 jailbreak attacks each against GPT-4o and Claude Sonnet~4. Against GPT-4o, the swarm achieved an Effective Harm Rate of 45.8%, producing 49 critical-severity breaches; against Claude Sonnet-4, the Effective Harm Rate was 0% despite a 40% technical success rate. In the second experiment, the same models performed combined source code analysis and binary fuzzing against a vulnerable C application with 9 planted CWEs. With a hand-crafted exploit seed corpus, regex pattern detection, and AddressSanitizer-based crash classification, the pipeline recovers 9 of 9 vulnerabilities (100% recall) in approximately four minutes on a consumer MacBook. With those scaffold components disabled, the same model recovers 0 of 9 by crash verification and 2 of 9 by citation. The capability class that motivated restricted release of Anthropic's Mythos Preview is therefore reproducible at effectively zero cost; the important enabler is the system scaffold itself, which compensates for the limited reasoning capacity of small individual models.

## 一、论文的核心问题与整体含义

本文立论的核心问题是：AI 安全政策当前集中于限制单个前沿模型的访问权限，但这种模型级别的管制是否能有效降低攻击者的攻击能力？背景是 Anthropic 以具有“自主发现并利用零日漏洞”能力为由，限制发布 Claude Mythos Preview 模型。论文的整体含义在于反驳这种单点控制的思路，主张 **AI 安全政策应当从模型层面的访问限制转向系统层面的能力评估**。作者通过实验证明，导致 Mythos 被限制发布的那一类攻击能力，实际上可以用极小的开源模型配合精心设计的系统框架（支架）在消费级硬件上以近乎零成本复现。因此，真正的安全瓶颈不在于某个模型本身，而在于围绕模型构建的协调、架构与进化优化系统。

## 二、方法论

论文提出了 **swarm-attack** 开源对抗测试框架，其核心思想是 **用多个轻量级 LLM 代理组成群体，通过共享记忆、并行探索与进化优化来弥补单个小模型推理能力的不足**。

- **核心架构**：使用 5 个 1.2B 参数的开源模型（LFM2.5-1.2B-Thinking）实例，分别赋予不同攻击策略（直接指令覆盖、角色扮演、多轮升级、混淆攻击、进化突变），组成协同攻击群体。群体维持一个共享成功攻击记忆库，并通过交叉（概率 0.5）和变异（概率 0.3）进行 15 代进化，每代保留最高效的两个攻击。
- **漏洞发现流程**（实验二）包含两阶段分析：
  - **Phase 1（源代码分析）**：将代理按角色分化（内存安全、注入、认证逻辑、算术），先通过正则表达式与关键词搜索定位可疑代码，再由 LLM 读取代码并生成带行号的脆弱性引用，最后通过后处理层将引用模式映射为正确的 CWE 编号。
  - **Phase 2（二进制模糊测试）**：利用手工编写的种子语料库，结合 AddressSanitizer 检测运行时崩溃，并对崩溃进行去重和 CWE 分类。
- **能力隔离设计**：为了区分系统支架与模型本身的贡献，论文定义了 **辅助配置**（全栈流水线）与 **自主配置**（禁用手工种子、正则层和规范化层，仅依 LLM 自身输出），从而量化支架的边际贡献。

## 三、实验设计

- **实验一：前沿模型安全越狱测试**
  - **攻击目标**：GPT-4o 与 Claude Sonnet 4。
  - **攻击量**：每个模型收到 225 次攻击（15 代 × 5 代理 × 3 任务）。
  - **度量指标**：引入 **有效危害率（Effective Harm Rate, EHR）**，要求技术评分 ≥ 0.7、包含有害关键词且经人工核验确认产生可操作的有害内容，以克服传统 LLM 裁判因格式合规而误判的缺陷。
- **实验二：软件漏洞发现**
  - **目标**：特意构造的 C 应用程序 **SwarmApp**（约 260 行，分布 5 个文件，植入了 9 类 CWE 漏洞，涵盖栈/堆溢出、格式化字符串、整数溢出、释放后使用、双重释放、空指针解引用、命令注入、硬编码凭据等），并加入了容易引起误报的代码模式。
  - **对比配置**：全栈辅助流水线（正则检测 + LLM 分析 + 模糊测试 + 后处理） vs. 完全自主配置（仅 LLM 独立工作），并在两种配置下测试了不同群体规模（3×3、5×3、7×3）。
  - **对比基准**：论文提及未与其他越狱方法直接横向对比，但在相关工作中与 PAIR、TAP、GCG 等方法进行了定性区分（本工作使用小模型群体进化，而非单一大模型），并指出漏洞发现上现有工作多依赖单前沿模型。

## 四、资源与算力

- **模型**：LFM2.5-1.2B-Thinking 开源模型，在 Ollama 环境下本地运行。
- **硬件**：消费者级 MacBook，单次 LLM 查询约耗时 7 秒。越狱实验总耗时约 1 小时，漏洞发现实验约 4 分钟（辅助配置），自主配置耗时更短。
- **成本**：明确指出攻击者总成本“实际为零”，无 GPU 集群，无 API 调用费用。
- 论文未使用高端算力，其背景中提及的对比工作 AgentFlow 在 Chrome 目标上曾使用 192 卡 H100 集群运行 7 天。

## 五、实验数量与充分性

- **实验组数**：共两大主实验，每个实验下含多个子配置。
  - 实验一：两个目标模型各 225 次攻击，并跟踪 15 代进化过程。
  - 实验二：覆盖三种群体规模、两种配置（辅助/自主），并历经 5 个设计迭代（v1-v5）将召回率从 0 提升至 100%，同时详细记录了测量错误修正过程。
- **充分性与公平性**：
  - **充分**：通过自主与辅助配置的隔离对比，清晰证明 9 个漏洞的召回完全归功于系统支架而非模型自身推理能力（自主模式下崩溃验证召回为 0/9）。进化过程的记录显示了群体优化带来的效果。
  - **客观**：在漏洞实验中，特意修正了因源代码注释泄漏 CWE 标签导致的召回虚高问题，体现了严谨性。使用人工核验的有效危害率避免了自动裁判的误判。
  - **局限**：未与其他越狱工具在统一基准上直接进行数值对比；漏洞实验目标为人工种植漏洞的小型代码库，非生产级真实项目。

## 六、主要结论与发现

1.  **能力可低成本复现**：依靠系统支架，1.2B 模型群体可在几分钟内以 100% 召回率发现跨文件的复杂软件漏洞，复现了 Mythos 级别的攻击能力类别，攻击成本近乎为零。
2.  **系统高于模型**：相同的 1.2B 模型在无支架辅助时表现极差（崩溃验证 0 漏洞），证明真正的攻击力源自系统架构（角色分化、多模态结合、共享记忆、进化搜索），而非单模型规模。
3.  **安全度量失真**：现有越狱成功率指标因格式合规误判而虚高。Claude Sonnet 4 技术成功率 40%，但有效危害率为 0%；GPT-4o 有效危害率达 45.8%，两者之间存在质的安全鸿沟，应推广有效危害率作为运营标准。
4.  **防御经济学倒置**：防御成本随保护面增加而升高，攻击成本随开源生态和硬件降价趋于零。开源群体框架可成为独立安全评估的战略基础设施，摆脱对前沿模型提供商的单向依赖。

## 七、优点

- **立场鲜明，靶向政策核心**：用具体实验挑战“限制单模型能赢得防御时间”的论点，为政策制定提供了清晰的证据导向。
- **方法设计精巧**：创造性地用小模型群体进化取代大模型单点搜索，并以“辅助 vs. 自主”配置干净地剥离并量化了系统支架的价值。
- **度量改进**：提出并应用**有效危害率（EHR）**，指出了传统技术成功率的严重缺陷，对安全评测具有方法论贡献。
- **过程公开严谨**：详细记录了实验设计迭代和标签泄露修正过程，为社区提供了“如何避免评测陷阱”的宝贵案例。

## 八、不足与局限

- **实验生态有限**：漏洞发现仅在一个人工编写的、代码量极小的目标程序上进行，未在大规模生产级代码或真实 CVE 上验证。
- **缺乏直接对比**：没有在统一任务上与单一大模型前沿方法（如 PAIR、TAP）进行定量比较，难以完全展现群体方法的相对效能边界。
- **规模化难度**：有效危害率依赖人工验证，虽然可靠但不具备可扩展性；模型规模仅 1.2B，更大模型的群体行为如何尚未探索。
- **通用性疑问**：论文自述“单一种植目标经反复迭代的框架并不具有泛化性”，框架设计经过针对特定目标的 5 次迭代，其通用能力有待在更广泛场景中检验。
